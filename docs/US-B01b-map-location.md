# US-B01b 地图选点扩展

## 交互与业务规则

- 活动创建页的“时间地点”步骤先选择城市，再选择城区、搜索地点或点击地图标记。
- 支持城市为杭州、北京；城市切换会同步切换地图中心、城区列表、推荐点位和活动列表。
- 搜索、地图点击和浏览器定位成功后，页面同时回填地址、城区、经度和纬度。
- 地图图层、地点搜索或地址反查不可用时，用户仍可直接填写完整地址；后端使用所选城市中心作为降级坐标，活动创建不被第三方地图服务阻断。
- 活动详情页展示城市、城区、完整地址、经纬度和单点地图。

## 接口

- `GET /api/map/places?keyword={keyword}&city={杭州|北京}`：地点搜索；高德未配置或异常时返回本地城市点位。
- `GET /api/map/reverse?longitude={lng}&latitude={lat}`：经纬度反查；异常时按最近城区降级。
- `GET /api/activities?city={杭州|北京}`：按城市查询已发布活动。
- 活动创建、草稿新增和草稿更新请求新增 `city`，并持久化 `city`、`district`、`longitude`、`latitude`。

## 北京支持

- 数据库迁移 `V6__beijing_city_and_map_points.sql` 增加活动城市字段，并初始化 4 场北京活动。
- 北京地图中心为 `39.904200, 116.407400`，包含东城、西城、朝阳、海淀、丰台等城区。
- 本地点位降级数据包含奥林匹克森林公园、国家图书馆、北京坊和 798 艺术区。
